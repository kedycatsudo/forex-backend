# 1.1.2 Env Contract & Validation Behavior

## Scope
This document defines runtime behavior for ingestion configuration loading and validation.

## Config source precedence
`IngestionSettings` is loaded via Pydantic settings with `env_file=".env"`.

Precedence order:
1. **Process environment variables** (highest priority)
2. **`.env` file values**
3. **Field defaults in `app/ingestion/config.py`** (lowest priority)

Implication:
- CI/CD or container-injected env vars override `.env`.
- Missing values fall back to defaults only when allowed by validation rules.

---

## Startup validation behavior (fail fast)
Validation runs when `IngestionSettings()` is instantiated (e.g., in startup hook).

Startup **fails immediately** if any rule is violated.

### Provider rules (A/B)
- `NEWS_PROVIDER_{X}_PROTOCOL` must be `ws` or `http`.
- If provider is enabled:
  - `NEWS_PROVIDER_{X}_URL` must be non-empty.
  - `NEWS_PROVIDER_{X}_API_KEY` must be non-empty.
- If protocol is `ws`:
  - `NEWS_PROVIDER_{X}_HEARTBEAT_INTERVAL_SECONDS` must be `> 0`.
- Reconnect rules:
  - `NEWS_PROVIDER_{X}_RECONNECT_BASE_SECONDS` must be `> 0`.
  - `NEWS_PROVIDER_{X}_RECONNECT_MAX_SECONDS` must be `>= RECONNECT_BASE_SECONDS`.
- Retry rules:
  - `NEWS_PROVIDER_{X}_MAX_RETRIES` must be `-1` (infinite) or `>= 0`.

### Provider rules (C/D, when enabled)
- `NEWS_PROVIDER_C_URL` and `NEWS_PROVIDER_C_API_KEY` required.
- `NEWS_PROVIDER_D_URL` and `NEWS_PROVIDER_D_API_KEY` required (current policy).

### Global rules
- `NEWS_INGESTION_LOOP_INTERVAL_SECONDS` must be `> 0`.

---

## Sanitized startup logging
On successful load, startup logs a sanitized summary from `ingestion_settings_summary()`:
- Includes booleans, names, protocol, url, intervals.
- Never logs raw secrets.
- Uses `api_key_set: true|false` instead of API key value.

## Common misconfigurations and outcomes

### 1) Enabled provider with missing URL
Example:
- `NEWS_PROVIDER_A_ENABLED=true`
- `NEWS_PROVIDER_A_URL=`
Outcome:
- Startup error: URL required when provider enabled.

### 2) Enabled provider with missing API key
Example:
- `NEWS_PROVIDER_B_ENABLED=true`
- `NEWS_PROVIDER_B_API_KEY=`
Outcome:
- Startup error: API key required when provider enabled.

### 3) Invalid protocol value
Example:
- `NEWS_PROVIDER_A_PROTOCOL=websocket`
Outcome:
- Startup error: protocol must be `ws` or `http`.

### 4) WS heartbeat invalid
Example:
- `NEWS_PROVIDER_A_PROTOCOL=ws`
- `NEWS_PROVIDER_A_HEARTBEAT_INTERVAL_SECONDS=0`
Outcome:
- Startup error: heartbeat must be > 0 for ws.

### 5) Reconnect max lower than base
Example:
- `NEWS_PROVIDER_A_RECONNECT_BASE_SECONDS=10`
- `NEWS_PROVIDER_A_RECONNECT_MAX_SECONDS=5`
Outcome:
- Startup error: reconnect max must be >= base.

### 6) Invalid max retries
Example:
- `NEWS_PROVIDER_B_MAX_RETRIES=-2`
Outcome:
- Startup error: retries must be `-1` or `>= 0`.

### 7) Invalid global loop interval
Example:
- `NEWS_INGESTION_LOOP_INTERVAL_SECONDS=0`
Outcome:
- Startup error due to `gt=0` constraint.

---

## Operational note
After any env change, restart the ingestion process to re-validate configuration on boot.
