# 1.1.9 Structured Logging Plan

## Required Log Fields (for key ingestion lifecycle events)

- `event` (string)
- `provider` (string | null)
- `connection_id` (string | null)
- `state` (string)
- `attempt` (int)
- `latency_ms` (int | null)
- `error_code` (string | null)
- `error_type` (string | null)
- `message_count` (int | null)
- `timestamp` (ISO8601 UTC)

## Standard Key Events

- `connect_start`
- `connect_ok`
- `connect_fail`
- `subscribe_ok`
- `subscribe_fail`
- `heartbeat_missed`
- `reconnect_scheduled`
- `reconnect_success`
- `message_received`

## Rules

- Always include required fields (null if unknown).
- Use stable event names (no variants like `connection_started`).
- Use UTC timestamps.
- Put provider-specific extras under `meta`.

## Why

Consistent fields enable fast filtering, dashboards, and post-incident timeline reconstruction.
